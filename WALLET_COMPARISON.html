Security Comparison

BFN Current State

  • Key in localStorage — accessible via DevTools, XSS, extensions
  • No encryption on stored key
  • No password protection
  • WIF displayed in DOM when user clicks "save your key"

Verdict: Acceptable for idea board with zero funds. Must upgrade before money flows.

Yours Wallet

  • Keys in extension storage — sandboxed from page scripts
  • Password required to unlock
  • However: keys may be extractable from browser console when extension is inspected
  • Extension developer tools can access stored data

Verdict: Better than raw localStorage, but still vulnerable to someone with DevTools/physical access.

Key Finding

Both systems are vulnerable to the same fundamental attack: someone with DevTools access can potentially extract the private key. The extension sandbox provides an extra layer, but it's not impenetrable.

The real security difference is smaller than expected. With passkey wrapping, BFN could potentially become more secure than Yours Wallet while maintaining zero-friction UX.

The BFN Advantage: Progressive Security

BFN's approach: Start with zero friction, upgrade security as stakes increase.

  • Day 1: Just post ideas (key in localStorage, no money involved)
  • When they care: "Keep your name" → backup download + PIN encryption
  • When money flows: Passkey wrapping (biometric) + fresh key rotation
  • At scale: Server HSM or threshold signing

Each upgrade is invisible to the user. Same name, same posts, same experience.

Integration Possibility

Future option: Power users who already have Yours Wallet could connect it via @1sat/connect. This gives them the full wallet experience while casual users stay on the zero-friction in-app system.

Both types of users coexist on the same platform, posting to the same board, with the same on-chain records.
