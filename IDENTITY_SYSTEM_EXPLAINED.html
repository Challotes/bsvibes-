The Code (Simplified)

async function createIdentity(): Promise { // 1. Generate cryptographically secure private key const { PrivateKey } = await import('@bsv/sdk'); const privateKey = PrivateKey.fromRandom(); // 2. Derive public key and address const publicKey = privateKey.toPublicKey(); const address = publicKey.toAddress(); // 3. Generate human-readable name const chars = '0123456789abcdefghijklmnopqrstuvwxyz'; let suffix = ''; for (let i = 0; i < 4; i++) { suffix += chars[Math.floor(Math.random() * chars.length)]; } const name = `anon_${suffix}`; // 4. Store locally const identity = { name, address: address.toString(), wif: privateKey.toWif() // Private key in WIF format }; localStorage.setItem('bfn_keypair', JSON.stringify(identity)); return identity; }

Security Model

Risks

⚠️ Key Loss

  • If user clears browser data without backing up, key is lost forever
  • Any funds at that address become inaccessible
  • Past posts remain, but user can't prove ownership

Mitigation: Clear prompts to save key. Future: server-encrypted backup option.

⚠️ Device Compromise

  • Malware could read localStorage and steal keys
  • Shared/public computers are risky

Mitigation: Users should only store significant value after moving to a proper wallet.

⚠️ No Password Protection

  • Anyone with physical access to the device can see the key
  • Browser extensions could potentially access localStorage

Mitigation: This is acceptable for small amounts. For larger amounts, users should export to a secure wallet.

How We Fund Posts (Server-Funded Model)
